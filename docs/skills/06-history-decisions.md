### 왜 폐기됐나

**근본 원인: 여성기업 자동 할인 오염**

`isWomenBiz`가 App.jsx L64에 디폴트 `true`로 하드코딩되어 있었음. UI에 토글이 없어서 사용자가 알 수 없었음. `eraFR()` 함수가 이 값을 받아 모든 예측에 -0.25%를 자동 적용.

영향:
- 5개월치 1,207건 예측 중 283건 직접 오염
- 하한율 데이터에 0.25% 누적 편향
- 모든 통계 모델이 잘못된 입력 학습
- 한전 경인 76억, LH 83억 같은 실제 낙찰 건의 floor_rate가 87.245%로 저장 (정상 89.745% 대신)

준의 결정: **"전면 백지화. 여성기업 할인 아예 없애기. 예측에서 완전 제거. 스프레드시트 사용."**

### 청산 절차 (Phase 11)

1. App.jsx의 `isWomenBiz` 13개 참조 모두 제거
2. utils.js의 `eraFR()` 시그니처 단순화 (3-param)
3. utils.js의 `womenBizAdj()` 함수 삭제
4. supabase.js의 7개 함수 stub 처리 (sbFetchScoring, sbFetchRoiMatrix 등)
5. DB에서 7개 의존 테이블 DROP

### 교훈

**1. 디폴트 값 변경의 폭발성**
중앙 변수의 디폴트가 `false → true`로 바뀌면 전체 시스템 영향. 변경 전 영향 분석 필수.

**2. UI 토글 없는 코드 변수의 위험성**
사용자가 인지할 수 없는 변수가 비즈니스 로직에 영향을 미치면 안 됨. 모든 전환 가능한 변수는 UI 노출.

**3. 정기적 무결성 검사 필요**
"왜 MAE가 안 줄어들지?" 같은 의문이 5개월간 해소되지 못함. 정기적 데이터 검증 + 백테스트 필요.

### 트레이드오프 발견: 진입률 vs MAE는 별개 지표
체크리스트의 "39.7% → 62.8% 진입률 개선"은 사실. 그러나 진입률(후보권 진입)과 사정률 MAE(가격 정밀도)는 **다른 지표**다:
- 진입률 ↑: v7이 1.6배 우수 (62.8% / 39.7%)
- MAE ↑↑: 군부대 +0.4353%p — 1순위 단독 낙찰의 1만~5만원 단위 정밀도가 무너짐
- 평균 언더컷 0.57%: 마진 손실의 단면

→ **둘 중 하나의 트레이드오프**가 아니라 **사정률 정확도가 무너진 채로 진입만 늘어나는 형태**여서 단독 낙찰률(목표 8%)은 오히려 후퇴할 위험.
